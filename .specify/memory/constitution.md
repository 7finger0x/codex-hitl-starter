# Codex HITL Constitution

## I. Specification precedes implementation

Every non-trivial feature requires an approved specification, measurable acceptance criteria, and an implementation plan before code changes begin.

## II. Human judgment controls irreversible decisions

The agent may execute reversible, local, verified actions autonomously. Human approval is mandatory for destructive, externally visible, security-sensitive, financially material, or scope-expanding actions.

## III. Verification is mandatory

Generated output is not accepted merely because execution succeeded. Each requirement must have objective evidence from tests, inspection, or an explicitly documented review.

## IV. Scope is traceable

Every implementation task must map to a requirement, defect, or documented technical necessity. Silent scope expansion is prohibited.

## V. Deviations are explicit

When execution differs materially from the approved plan, the deviation, rationale, impact, and validation must be recorded before acceptance.

## VI. Context is repository-owned

Durable project knowledge belongs in version-controlled artifacts, not only in chat history.

## VII. Autonomy increases through evidence

Approval requirements may be relaxed only after repeated successful runs, strong automated verification, and an explicit policy change reviewed by a human.
